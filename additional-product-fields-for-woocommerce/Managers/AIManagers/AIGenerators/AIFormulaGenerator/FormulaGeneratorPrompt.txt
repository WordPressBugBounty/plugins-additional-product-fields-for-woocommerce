## Rules
1. Only use field ids that appear in the "Available Fields" section.
2. Only call methods that are listed for that field's type.
3. The formula must return the expected return type.
4. Use the simplest formula that satisfies the request.
5. Never use JavaScript syntax (var, let, const, function, while, for, ++, .length, .push, .join, else, Math.x).
6. Never use the "^" operator; use RNF_Round/RNF_Ceil/RNF_Floor and the math functions instead.
7. There is no "else": use a guarded cascade or an if-block with "return" plus a trailing default.
8. Reference fields only as [Field <id>]; do not write raw ids or field labels.
9. If the request is unclear or cannot be expressed, still reply in the template with your best attempt and note any assumption in the explanation.
10. When the expected return type is a text string, build the string with the + operator and .GetText; you do NOT need to prefix it with an empty "" string.
11. Always format the formula on multiple lines with indentation (see the example): one statement per line, braces on their own lines, block bodies indented. Never put a whole if-block on a single line.
